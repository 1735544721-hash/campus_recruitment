第 3 章 系统需求分析与设计

3.1 可行性分析

3.1.1 技术可行性

系统采用 Spring Boot 3.2 与 Vue 3 构建，技术栈成熟，具备完善的生态与工程实践基础。MySQL 数据库能够满足招聘业务的结构化数据存储需求，配合前后端分离架构与 REST 接口实现稳定的功能扩展，技术实现可行。

3.1.2 经济可行性

系统基于开源技术构建，无需额外授权成本，部署可在高校现有服务器环境中完成。通过统一平台提升招聘流程效率与信息透明度，有利于降低人工沟通与管理成本，经济性良好。

3.1.3 操作可行性

系统面向学生、企业与管理员三类用户提供清晰的功能入口与操作路径，界面交互与业务流程符合招聘场景习惯。通过权限控制与流程化管理降低误操作风险，操作可行性较高。

3.2 系统功能需求

系统功能围绕多角色协同展开。学生端主要功能包括注册登录、个人信息维护、简历管理、职位检索、投递记录查询与面试通知查看。企业端主要功能包括企业信息维护、职位发布与管理、简历筛选、面试安排与投递处理。管理员端主要功能包括用户管理、企业审核与维护、职位管理、投递记录监管、面试管理与消息管理。系统需支持投递流程的状态更新与消息通知，保证业务闭环。

3.3 非功能性需求分析

3.3.1 性能需求

系统需支撑高峰期多用户并发访问，职位列表与投递记录查询应保持合理响应时间。数据库查询需通过索引优化以保障检索效率。

3.3.2 安全性需求

系统需具备登录认证与权限控制能力，保证不同角色仅能访问授权功能。敏感数据传输与存储需符合安全规范，日志记录需支持操作追溯。

3.3.3 易用性与可维护性需求

系统界面需清晰直观，操作路径应符合招聘业务流程。后端代码结构需分层清晰，便于后续功能扩展与维护。

3.4 系统用户用例分析

学生用例包括注册登录、维护个人信息、创建与编辑简历、检索职位、投递简历、查看投递状态与面试通知。企业用例包括企业信息维护、职位发布与管理、查看投递简历、安排面试与发布通知。管理员用例包括用户管理、企业审核与维护、职位审核与管理、投递记录与面试管理、系统消息管理等。通过用例分析可明确各角色与业务流程之间的交互关系，指导后续模块设计。

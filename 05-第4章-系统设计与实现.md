第 4 章 系统设计与实现

4.1 功能模块设计

校园招聘系统按角色划分为学生端、企业端与管理员端三类功能体系。学生端模块包含注册登录、简历管理、职位检索与投递记录、面试通知与消息查看等功能，保证求职流程完整可追溯；企业端模块包含企业信息维护、职位发布与管理、简历筛选与投递处理、面试安排与反馈等功能，形成招聘闭环；管理员端模块涵盖用户管理、企业管理、职位管理、投递记录监管、面试管理与消息管理，负责平台运行与数据监管。各模块边界清晰、数据互通，支撑校园招聘资源整合与高效协同。

4.2 数据库设计

4.2.1 E-R 设计

系统核心实体包括用户、学生、企业、职位、简历、投递记录、面试与消息。用户实体用于统一账号体系，学生与企业实体承载角色信息；职位与简历形成投递关系，投递记录与面试安排构成业务链路，消息实体承载通知与反馈。实体之间通过主键与外键建立关联，保证招聘流程的数据一致性与可追溯性。

4.2.2 逻辑设计

系统核心数据表围绕用户、业务与交互三类场景设计，不同类别数据表有明确架构与核心功能：用户类表（用户表、学生表、企业表）存储角色信息，业务类表（职位表、简历表、投递记录表、面试表）承载核心服务数据，交互类表（消息表）记录通知与反馈。下文按类别阐述各核心数据表的构建过程，确保数据模型贴合系统功能需求。

4.3 前端功能实现

4.3.1 职位列表页面及功能

职位列表页面提供关键词检索与条件筛选，支持查看职位详情、岗位要求与企业信息，帮助学生快速定位匹配岗位。列表支持分页展示，保证在岗位数量较多时仍保持良好响应。

4.3.2 简历管理页面及功能

简历管理页面支持创建、编辑与预览简历，学生可维护教育经历、技能与项目经验，并基于简历进行岗位投递，保证投递信息完整一致。

4.3.3 投递记录页面及功能

投递记录页面展示投递状态与处理进度，支持按岗位与状态筛选，便于学生跟踪投递流程与企业反馈。

4.3.4 企业端职位管理页面及功能

企业端职位管理页面支持岗位新增、编辑、下线与状态管理，并可查看投递简历列表与候选人信息，提升招聘管理效率。

4.3.5 面试与消息通知页面及功能

面试管理页面用于安排面试时间与地点，支持面试状态更新；消息通知页面用于展示系统通知与招聘反馈，保证沟通及时准确。

4.4 后端功能实现

4.4.1 用户登录与权限控制

后端提供登录认证接口，结合 JWT 生成登录令牌，并根据角色信息实现接口访问控制。

```java
@PostMapping("/api/auth/login")
public Result login(@RequestBody LoginRequest request) {
    if (StringUtils.isBlank(request.getUsername()) || StringUtils.isBlank(request.getPassword())) {
        return Result.error("用户名或密码不能为空");
    }
    User user = userService.validate(request.getUsername(), request.getPassword());
    if (user == null) {
        return Result.error("用户名或密码错误");
    }
    String token = tokenService.generate(user.getId(), user.getRole());
    return Result.ok(new LoginResponse(token, user));
}
```

4.4.2 企业与学生信息管理接口

企业与学生管理接口用于新增、修改与查询角色信息，保证用户资料完整一致。

```java
@PostMapping("/api/company")
public Result createCompany(@RequestBody CompanyRequest request) {
    companyService.create(request);
    return Result.ok();
}
```

4.4.3 职位发布与管理接口

职位发布接口用于企业新增职位信息，系统校验岗位名称、薪资与地点等必填字段后写入数据库；职位管理接口用于更新岗位状态与信息。

```java
@PostMapping("/api/positions")
public Result createPosition(@RequestBody PositionRequest request) {
    if (!request.isValid()) {
        return Result.error("职位信息不完整");
    }
    Long id = positionService.create(request);
    return Result.ok(id);
}
```

4.4.4 简历管理与投递接口

简历接口支持学生创建与更新简历，投递接口用于生成投递记录并同步更新投递状态，保证招聘流程可追溯。

```java
@PostMapping("/api/applications")
public Result apply(@RequestBody ApplyRequest request) {
    if (!resumeService.exists(request.getResumeId())) {
        return Result.error("简历不存在");
    }
    applicationService.create(request);
    return Result.ok();
}
```

4.4.5 面试安排与消息通知接口

面试接口支持企业安排面试并记录时间地点，消息接口用于向学生与企业发送通知并记录消息内容。

```java
@PostMapping("/api/interviews")
public Result schedule(@RequestBody InterviewRequest request) {
    interviewService.create(request);
    messageService.sendInterviewNotice(request.getStudentId(), request.getInterviewId());
    return Result.ok();
}
```

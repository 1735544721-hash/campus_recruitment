第 4 章 系统设计与实现

4.1 功能模块设计

校园招聘系统按角色划分为学生端、企业端与管理员端三类功能体系。学生端模块包括注册登录、个人信息维护、简历管理、职位检索与投递记录、面试通知与消息查看等功能，保证求职流程完整可追溯。企业端模块包括企业信息维护、职位发布与管理、简历筛选与投递处理、面试安排与通知管理，形成招聘业务闭环。管理员端模块涵盖用户管理、企业管理、职位管理、投递管理、面试管理与消息管理，负责平台运行与数据监管。各模块边界清晰，数据通过统一接口流转，支撑多角色协同运行。

4.2 数据库设计

4.2.1 E-R 设计

系统核心实体包括用户、学生、企业、职位、简历、投递记录、面试与消息。用户实体用于统一账号体系，学生与企业实体承载角色信息；职位与简历形成投递关系，投递记录与面试安排形成业务链路，消息实体承载通知与反馈。实体之间通过主键与外键建立关联，保证招聘流程的数据一致性与可追溯性。

4.2.2 逻辑设计

系统数据表围绕用户、业务与交互三类场景组织。用户类表包含用户表、学生表与企业表，记录账号信息与角色资料；业务类表包含职位表、简历表、投递记录表与面试表，承载招聘流程数据；交互类表包含消息表，用于记录通知与反馈。各表通过主键与索引保证查询效率，为投递筛选、面试安排与消息推送提供可靠的数据支撑。

4.3 前端功能实现

4.3.1 首页与职位列表

首页展示热门职位与企业信息，职位列表提供关键词检索与条件筛选，支持查看职位详情与岗位要求，保证学生能够快速定位匹配岗位。

4.3.2 简历管理与投递记录

简历管理页面支持创建、编辑与预览简历，学生可基于简历信息进行岗位投递。投递记录页面展示投递状态与处理进度，便于学生跟踪求职流程。

4.3.3 企业信息与职位管理

企业端提供企业信息维护与职位发布入口，支持职位新增、编辑与状态管理，并可查看投递简历与候选人信息，提升招聘管理效率。

4.3.4 面试与消息通知

面试管理页面用于安排面试时间与地点，并支持面试状态更新。消息通知页面用于展示系统通知与招聘反馈，保证沟通及时准确。

4.4 后端功能实现

4.4.1 用户登录与权限控制

后端提供登录认证接口，结合 Spring Security 完成角色权限校验。登录成功后生成令牌并返回用户信息，用于后续接口鉴权。

```java
@PostMapping("/api/auth/login")
public Result login(@RequestBody LoginRequest request) {
    if (StringUtils.isBlank(request.getUsername()) || StringUtils.isBlank(request.getPassword())) {
        return Result.error("用户名或密码不能为空");
    }
    User user = userService.validate(request.getUsername(), request.getPassword());
    if (user == null) {
        return Result.error("用户名或密码错误");
    }
    String token = tokenService.generate(user.getId(), user.getRole());
    return Result.ok(new LoginResponse(token, user));
}
```

4.4.2 职位发布与管理接口

职位发布接口用于企业新增职位信息，系统校验岗位名称、薪资与地点等必填字段后写入数据库；职位管理接口用于更新岗位状态与信息。

```java
@PostMapping("/api/positions")
public Result createPosition(@RequestBody PositionRequest request) {
    if (!request.isValid()) {
        return Result.error("职位信息不完整");
    }
    Long id = positionService.create(request);
    return Result.ok(id);
}
```

4.4.3 简历管理与投递接口

简历接口支持学生创建与更新简历，投递接口用于生成投递记录并同步更新投递状态，保证招聘流程可追溯。

```java
@PostMapping("/api/applications")
public Result apply(@RequestBody ApplyRequest request) {
    if (!resumeService.exists(request.getResumeId())) {
        return Result.error("简历不存在");
    }
    applicationService.create(request);
    return Result.ok();
}
```

4.4.4 面试安排与消息通知接口

面试接口支持企业安排面试并记录时间地点，消息接口用于向学生与企业发送通知并记录消息内容。

```java
@PostMapping("/api/interviews")
public Result schedule(@RequestBody InterviewRequest request) {
    interviewService.create(request);
    messageService.sendInterviewNotice(request.getStudentId(), request.getInterviewId());
    return Result.ok();
}
```

4.4.5 投递状态与数据统计接口

投递状态接口用于更新投递进度，统计接口用于输出投递数量、职位热度等指标，为后台管理提供数据支撑。

```java
@PutMapping("/api/applications/{id}/status")
public Result updateStatus(@PathVariable Long id, @RequestParam String status) {
    applicationService.updateStatus(id, status);
    return Result.ok();
}
```
